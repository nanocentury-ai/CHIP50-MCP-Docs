What is this?

A research tool that speaks plain English

CHIP50 is a large-scale, nationally representative panel survey that has been tracking how Americans use social media — and how that use connects to their politics, demographics, and mental health — across more than 20 survey waves since 2020. Each wave interviews roughly 10,000 respondents, weighted to match the U.S. population.

This project gives you a way to talk to that data using Claude AI. Instead of writing SQL queries or running statistical software, you just describe what you want to know. Claude figures out which tools to call, runs the appropriate queries against the BigQuery database, and returns tables, charts, and summaries — all properly weighted for population representativeness.

Think of it as having a research assistant who has read every row of the dataset and can instantly produce crosstabs, trend lines, regression tables, and prose summaries on demand.

The system uses the Model Context Protocol (MCP), an open standard that lets AI assistants call external tools. The CHIP50 MCP server runs in the cloud (Google Cloud Run) and exposes 14 specialized analysis tools that Claude can invoke automatically as it answers your questions.

How it works

The Data

What's in the dataset

CHIP50 covers social media behavior across 232 variables spanning platforms, demographics, politics, mental health, and media consumption.

Platforms (19)

  • Facebook, Instagram, X/Twitter
  • TikTok, YouTube, Snapchat
  • LinkedIn, Reddit, WhatsApp
  • Pinterest, Messenger, Tumblr
  • Truth Social, Gab, Parler
  • Mastodon, Bluesky, 4chan
  • Usage, frequency, trust, posting

Demographics (9)

  • Age (8-category)
  • Gender
  • Race/Ethnicity (5-category)
  • Education level
  • Income (10-category)
  • Party ID (3- and 7-point)
  • Urban/suburban/rural type
  • U.S. State (all 50)

Mental Health

  • Full PHQ-9 scale (9 items)
  • Depressive symptom severity
  • Suicidal ideation item
  • Sleep, concentration, energy
  • Anhedonia, hopelessness
  • Guilt, appetite, psychomotor
  • Clinical cutoff scoring

Political Attitudes

  • Ideology (7-point scale)
  • Voting behavior
  • Institutional trust (6 items)
  • Trust in science, government
  • Trust in media, tech platforms
  • Political news sources
  • Political posting behavior

Waves & Coverage

  • Waves 14–35+ (ongoing)
  • ~10,000 respondents/wave
  • Nationally representative
  • Survey weights provided
  • Panel design (repeat measures)
  • Wave-specific date ranges
  • Trend analysis across time

Privacy Protections

  • Aggregated data only
  • No individual records returned
  • Cell suppression (n < 30)
  • Sensitive items (PHQ-9): aggregate only
  • Sentinel values filtered (-99)
  • No PII stored
  • OAuth authentication
Sample Queries

What you can ask

From a quick one-liner to a full research paper — here are examples of what the system can handle, roughly in order of complexity.

🗺️
Suicidal Ideation by State
Quick
Generate a choropleth map showing variation in suicidal ideation (PHQ-9 item 9) prevalence across all 50 states.
generate_crosstab generate_marginals
Create a choropleth map showing variation in suicidal ideation at the state level. Use the PHQ-9 item 9 data and display results as a percentage of respondents who reported any suicidal ideation (score ≥ 1).
What you get
A population-weighted state-level table of PHQ-9 item 9 prevalence (suppressed for states with n < 30), ready to be visualized as a choropleth map using Claude's built-in code generation or pasted into mapping software like Datawrapper or Flourish. States with highest and lowest rates are highlighted in a brief summary.
📊
TikTok Adoption by Age and Party
Quick
Cross-tabulate TikTok usage against age group and political party in a single wave to see who's on the platform.
generate_crosstab_batch
Show me TikTok adoption rates broken down by age group and by political party in the most recent wave. Format as a table with percentages.
What you get
Two population-weighted tables showing what share of each age group and each partisan group uses TikTok, with sample sizes and confidence intervals. The tool runs both crosstabs in parallel for efficiency.
📈
Twitter/X Usage Trend Since 2022
Quick
Plot the trajectory of Twitter/X adoption across all survey waves, including the period surrounding the Musk acquisition.
get_platform_trends
Show me how Twitter/X usage has changed across all available survey waves. Plot the trend and note any sharp increases or decreases.
What you get
A wave-by-wave time series of Twitter/X adoption rates with exact dates, rendered as a trend line with percentage-point changes between waves flagged for significant shifts.
💬
Institutional Trust Snapshot
Quick
Compare trust levels in science, government, social media companies, and specific platforms across partisan groups.
get_ordinal_crosstab generate_marginals_batch
Compare trust in science, trust in social media companies, and trust in Facebook and Twitter broken down by political party. Use the most recent wave.
What you get
Mean trust scores (1–4 scale) for each institution by partisan group, arranged in a comparison table with statistically meaningful differences highlighted.
🏆
Demographic Profiles of the Top 10 Platforms Over Time
Medium
For each of the 10 most-used platforms, describe the full demographic profile (age, gender, race, income, education, party) and show how it has shifted across waves.
get_platform_posting_summary generate_crosstab_batch get_platform_trends
What are the demographic profiles of the top 10 social media platforms by usage, and how have those profiles changed over time? For each platform, show breakdowns by age, gender, race/ethnicity, income, education, and party identification, and track how these have shifted across survey waves. Where possible, note how the CHIP50 data aligns with or diverges from publicly available platform demographic data from Pew Research or platform-disclosed figures.
What you get
A structured report with one section per platform. Each section includes a demographic heatmap table for the most recent wave, a trend chart of the platform's overall adoption trajectory, a brief narrative profile ("TikTok skews younger, female, and slightly more Democratic, with the youngest cohort gap narrowing since Wave 18…"), and a note on how these patterns compare to published third-party research.
🔀
Platform Migration After the Musk Acquisition
Medium
Examine whether Twitter/X users migrated to alternative platforms (Mastodon, Bluesky, Threads) after October 2022, and which demographic groups drove that shift.
get_platform_trends generate_crosstab_filtered run_logistic_regression
Analyze platform migration away from Twitter/X following the Musk acquisition. Show trends in Twitter/X usage across waves, alongside adoption trends for Mastodon, Bluesky, and other alternatives. Break down which demographic and partisan groups drove any observed decline. Run a logistic regression predicting Twitter/X abandonment as a function of demographics and political affiliation.
What you get
A side-by-side trend analysis of Twitter/X vs. alternative platforms, a demographic breakdown of who left and who stayed, and a logistic regression table showing odds ratios for platform abandonment by demographic group — with a plain-English summary of the key findings.
🧠
Political Echo Chambers by Platform
Medium
Estimate the political composition of each platform's user base and examine how ideological clustering has changed over time.
generate_crosstab_batch get_platform_trends get_ordinal_crosstab
For each major social media platform, show the partisan composition of the user base (% Democrat, % Republican, % Independent) in the most recent wave. Then show how ideological balance has shifted across waves for the five platforms with the most partisan skew. Also show mean ideology scores (7-point scale) for users of each platform.
What you get
A ranked table of platforms by partisan skew, with composition percentages for each party, plus trend lines for the most polarized platforms, and a mean ideology score comparison across all platforms — illustrating which platforms have become more or less ideologically sorted over time.

Full Research Paper: Social Media Use and Depression
Deep
Generate a complete academic paper — hypotheses grounded in theory, a research design section, empirical tests with regression tables, and a discussion with citations.
run_ols_regression run_logistic_regression get_platform_trends generate_crosstab_batch get_ordinal_distribution
Write a research paper on social media use and depression using the CHIP50 data. The paper should have four sections: 1. **Hypotheses**: Lay out a set of theoretically grounded hypotheses about the relationship between social media use and depression, drawing on the existing literature (displacement theory, social comparison theory, passive vs. active use, platform-specific effects, moderation by demographics). 2. **Research Design**: Describe how to test each hypothesis using the available CHIP50 variables, specifying the outcome measure (PHQ-9 score or clinical cutoff), the key predictors, relevant controls, and the appropriate statistical approach (OLS for continuous PHQ-9, logistic for clinical cutoff). 3. **Results**: Test each hypothesis using the data. Produce regression tables for each model with coefficients, standard errors, and significance levels. For each hypothesis, write a brief paragraph interpreting whether it is supported, partially supported, or rejected. 4. **Discussion and Conclusion**: Summarize the key findings, note limitations (cross-sectional design, self-report, platform measures), and situate the results in the broader literature. Provide a reference list.
What you get
A full draft research paper (~3,000–5,000 words) with: a literature-grounded theory section articulating 6–10 testable hypotheses; a methods section with variable operationalization and model specifications; actual regression tables run against the CHIP50 data (with survey weights and proper reference category selection); hypothesis-by-hypothesis results writeups; a discussion addressing limitations and future directions; and an APA-format reference list drawing on published social media and mental health research. The paper is structured and formatted to be used as a working draft or teaching example.

Social Media and Political Polarization
Deep
Investigate whether heavy social media use is associated with more extreme political attitudes, and whether this effect varies by platform and demographic group.
run_ols_regression generate_crosstab_filtered get_ordinal_crosstab get_platform_trends
Investigate whether social media use predicts more extreme ideological positions. Using ideology (7-point scale) as the outcome variable, run OLS regressions with platform usage frequency as predictors, controlling for demographics. Test whether the relationship differs by platform (e.g., does heavy Twitter/X use predict more extreme ideology differently than heavy Facebook use?). Also run interaction models to test whether the effect of social media use on ideology is moderated by age, education, or party ID. Produce a full set of regression tables and a narrative summary.
What you get
A series of OLS regression models — a baseline demographic model, platform-specific models, and interaction models — with tables showing platform-by-platform coefficients for ideology, followed by a narrative discussing which platforms show the strongest associations and how those effects vary across demographic groups.
🌐
The Geography of Social Media Use
Deep
A comprehensive geographic analysis of social media adoption, usage intensity, and political posting across U.S. states and urban/rural typology.
generate_crosstab generate_crosstab_filtered run_ols_regression get_freq_trends
Conduct a comprehensive geographic analysis of social media use. First, produce state-level adoption rates for the five largest platforms and identify regional patterns. Second, examine how urban/rural typology mediates platform choice — do rural users cluster on different platforms than urban users? Third, analyze political posting behavior geographically: which states and urban/rural categories show the highest rates of political posting on social media? Fourth, run a regression predicting frequency of political posting with state-level partisanship (proxied by party ID), urban type, age, and income as predictors. Generate choropleth-ready data tables for each major finding.
What you get
State-level data tables for each platform (choropleth-ready), an urban/rural platform preference analysis, a political posting geography table, and a regression model of political posting determinants — all with a structured narrative summary connecting the geographic patterns.
Tools Reference

What Claude can call

Claude automatically selects the right tools for your question. Here's what's available under the hood.

get_available_variables

Variable Discovery

Returns all available columns, their categories (demographic, platform, PHQ-9, etc.), and the range of waves in the dataset. Claude calls this first to orient itself to the data.

generate_marginals

Single-Variable Distribution

Returns the population-weighted distribution of a single variable — e.g., what share of respondents use Facebook, or how PHQ-9 scores are distributed.

generate_marginals_batch

Batch Distributions

Runs marginal distributions for multiple variables in parallel. Useful when you want a snapshot of many platforms or survey items at once.

generate_crosstab

Platform × Demographic Crosstab

Produces adoption rates for one platform broken down by one demographic variable (age, gender, party, etc.) in a given wave, with counts and weighted percentages.

generate_crosstab_filtered

Filtered Crosstab

Like generate_crosstab, but allows additional demographic filters to analyze subgroups — e.g., TikTok usage by age, among women only.

generate_crosstab_batch

One Platform, Many Demographics

Runs crosstabs for one platform across multiple demographic dimensions simultaneously. Efficient for building a complete demographic profile.

get_platform_trends

Adoption Trends Over Time

Returns wave-by-wave adoption rates for a platform, optionally broken down by a demographic group, enabling time-series analysis.

get_freq_trends

Usage Frequency Trends

Tracks mean usage frequency (1–6 ordinal scale) for a platform across waves, capturing not just whether people use a platform but how intensely.

get_ordinal_distribution

Ordinal Variable Distribution

Returns the full distribution of ordinal variables: usage frequency, trust scales (1–4), political ideology (1–7), and PHQ-9 item scores.

get_ordinal_crosstab

Mean Scores by Demographic

Computes mean ordinal scores broken down by demographic group — e.g., mean PHQ-9 total by age, or mean platform trust by party.

get_platform_posting_summary

Full Platform Summary

Returns all key metrics for one platform in a single call: adoption rate, usage frequency distribution, trust score, and political posting behavior.

run_ols_regression

OLS Regression

Runs population-weighted OLS regression with user-specified outcome, predictors, and reference categories. Outputs coefficients, standard errors, t-statistics, p-values, and R².

run_logistic_regression

Logistic Regression

Runs population-weighted logistic regression for binary outcomes (e.g., platform adoption, clinical depression cutoff). Returns odds ratios and 95% confidence intervals.

introduce_mcp

Server Introduction

Returns a plain-English overview of the MCP server, available tools, and the structure of the dataset. A good starting point when exploring the system.

Setup

Get it running in 15 minutes

The server runs on Google Cloud Run and connects to BigQuery. You need a Google Cloud account and Claude Desktop (or Claude.ai with MCP support).

Prerequisites

  • ✓
    Google Cloud account with billing enabled ($6–15/mo estimated)
  • ✓
    gcloud CLI — brew install google-cloud-sdk on macOS
  • ✓
    Python 3.11+ and uv package manager
  • ✓
    Claude Desktop (free) or any MCP-compatible client
  • ✓
    CHIP50 data loaded into BigQuery (handled by load_data.sh)
1
Clone the repo and set your project
git clone https://github.com/nanocentury-ai/DDFchip50
cd DDFchip50/remote-mcp
export GCP_PROJECT="your-gcp-project-id"
2
Authenticate and deploy
gcloud auth login
gcloud auth application-default login
./load_data.sh   # loads CHIP50 CSV into BigQuery
./deploy.sh      # builds + deploys to Cloud Run
3
Configure Claude Desktop

Add to ~/Library/Application Support/Claude/claude_desktop_config.json:

{
  "mcpServers": {
    "chip50": {
      "url": "https://YOUR-SERVICE.run.app/mcp"
    }
  }
}
4
Restart Claude Desktop and verify
What variables are available in the CHIP50 dataset?

You should see the full list of platforms, demographics, and wave range.
